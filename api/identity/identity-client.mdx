## Overview

The `IdentityClient` provides access to identity management functionality including user profiles, aliases, and origin management.

```typescript
import { SDK } from '@bloque/sdk';

const bloque = new SDK({
  origin: 'your-origin',
  auth: {
    type: 'apiKey',
    apiKey: 'your-api-key'
  },
  mode: 'production'
});

const session = await bloque.connect('user@example.com');
const identity = session.identity;
```

## Properties

### aliases

<ResponseField name="aliases" type="AliasesClient">
  Client for managing user aliases (phone numbers, emails, etc.)
  
  See [AliasesClient](/api/identity/aliases-client) for details.
</ResponseField>

### origins

<ResponseField name="origins" type="OriginsClient">
  Client for managing identity origins and registration
  
  See [OriginsClient](/api/identity/origins-client) for details.
</ResponseField>

## Methods

### me()

Retrieve the authenticated user's identity information including profile data, status, and metadata.

```typescript
const identity = await bloque.identity.me();
console.log(identity.urn); // "did:bloque:origin:..."
console.log(identity.profile.email); // User's email
```
